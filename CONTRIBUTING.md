# Contributing

Small, focused pull requests are welcome.

1. Fork the repository and create a branch.
2. Keep credentials and personal media out of commits.
3. Run `make validate`.
4. Update `CHANGELOG.md` when behavior changes.
5. Explain how the change was tested in the pull request.

The project intentionally avoids a JavaScript build step and database so it stays
easy to audit and deploy on ordinary Apache hosting.
